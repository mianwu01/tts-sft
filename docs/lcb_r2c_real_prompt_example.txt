### Approach
1. **Problem Analysis**: For each index `i`, we need to consider all indices `j` such that `nums1[j] < nums1[i]` and select at most `k` values from `nums2[j]` to maximize the sum. The challenge is to do this efficiently for large `n`.

2. **Intuition**: By sorting the indices based on `nums1` values, we can process indices in increasing order of `nums1`. For each group of indices with the same `nums1` value, we first compute the answer for all indices in the group using the current top `k` values from previously processed groups (which have smaller `nums1` values). Then, we add the current group's `nums2` values to a min-heap to maintain the top `k` values for subsequent groups.

3. **Algorithm Selection**: 
   - **Sorting**: We sort the indices based on `nums1` to process them in increasing order.
   - **Group Processing**: Using `itertools.groupby`, we group indices with the same `nums1` value. For each group, we first set the answer for all indices in the group to the current sum of top `k` values (from the heap).
   - **Min-Heap**: A min-heap is used to efficiently track the top `k` values from `nums2` of processed indices. The heap allows us to maintain the smallest value among the top `k` values, facilitating quick updates when a larger value is encountered.

4. **Complexity Analysis**: 
   - **Sorting**: O(n log n) time.
   - **Group Processing**: O(n) time since each element is processed once.
   - **Heap Operations**: Each insertion and extraction operation is O(log k), leading to O(n log k) time for all heap operations.
   - **Overall Complexity**: O(n log n + n log k) = O(n log n) since `k` ≤ `n`. This is efficient for the given constraints (n ≤ 10^5).

### Solution Code
```python
import heapq
from itertools import groupby
from typing import List

class Solution:
    def findMaxSum(self, nums1: List[int], nums2: List[int], k: int) -> List[int]:
        n = len(nums1)
        arr = [(nums1[i], nums2[i], i) for i in range(n)]
        arr.sort(key=lambda x: x[0])
        
        heap = []
        total = 0
        ans = [0] * n
        
        for key, group in groupby(arr, key=lambda x: x[0]):
            for _, _, idx in group:
                ans[idx] = total
            
            for _, num2, _ in group:
                if len(heap) < k:
                    heapq.heappush(heap, num2)
                    total += num2
                else:
                    if num2 > heap[0]:
                        removed = heapq.heappop(heap)
                        total -= removed
                        heapq.heappush(heap, num2)
                        total += num2
        return ans
```

### Explanation
1. **Initialization**: We start by creating a list of tuples `(nums1[i], nums2[i], i)` for each index `i`.
2. **Sorting**: This list is sorted based on `nums1[i]` to process indices in increasing order of `nums1` values.
3. **Group Processing**: Using `itertools.groupby`, we process indices in groups where each group contains indices with the same `nums1` value. For each group:
   - **Answer Assignment**: The current sum of the top `k` values (from the heap) is assigned to all indices in the group. This sum represents the maximum sum of at most `k` values from `nums2` at indices where `nums1[j] < nums1[i]`.
   - **Heap Update**: The `nums2` values from the current group are added to the min-heap. If the heap size is less than `k`, the value is added directly. Otherwise, if the value is larger than the smallest value in the heap, the smallest value is replaced with the new value, updating the total sum accordingly.
4. **Result Construction**: After processing all groups, the `ans` array contains the computed results for each index, which is returned as the solution.

This approach efficiently handles the constraints by leveraging sorting and a min-heap to maintain and update the top `k` values dynamically, ensuring optimal performance for large input sizes.

### Approach
1. **Sorting by `nums1`**: We first create a list of tuples containing `(nums1[i], nums2[i], i)` for each index `i`. This list is sorted in ascending order based on `nums1[i]`. 2. **Min-Heap for Top `k` Elements**: As we process each element in the sorted list, we maintain a min-heap of size at most `k`. The heap helps efficiently track the top `k` largest values of `nums2` encountered so far (from elements with smaller `nums1` values).

```python
import heapq
from typing import List

class Solution:
    def findMaxSum(self, nums1: List[int], nums2: List[int], k: int) -> List[int]:
        n = len(nums1)
        arr = [(nums1[i], nums2[i], i) for i in range(n)]
        arr.sort(key=lambda x: x[0])
        
        heap = []
        current_sum = 0
        ans = [0] * n
        
        for num1, num2, idx in arr:
            ans[idx] = current_sum
            
            if len(heap) < k:
                heapq.heappush(heap, num2)
                current_sum += num2
            else:
                if num2 > heap[0]:
                    current_sum -= heap[0]
                    heapq.heappop(heap)
                    heapq.heappush(heap, num2)
                    current_sum += num2
        
        return ans
```

To solve this problem, we need to efficiently compute for each index `i` in the arrays `nums1` and `nums2` the maximum sum of at most `k` values from `nums2` at indices `j` where `nums1[j] < nums1[i]`. Given the constraints (with `n` up to 100,000), a brute-force approach would be infeasible, so we use a combination of sorting and a min-heap to optimize the solution.

### Approach
1. **Sorting by `nums1`**: We first create a list of tuples `(nums1[i], nums2[i], i)` for each index `i`, then sort this list by `nums1` in ascending order. This allows us to process elements in increasing order of `nums1`, ensuring that for any element `i`, all elements with `nums1[j] < nums1[i]` have already been processed.
2. **Processing Groups**: We process elements in groups where each group consists of indices with the same `nums1` value. For each group:
   - **Set Answers**: For each index in the current group, the answer is the current sum of the top `k` values from `nums2` of elements processed so far (which have `nums1` strictly less than the current group's `nums1`).
   - **Update Heap**: We then add the `nums2` values of the current group to a min-heap. The heap maintains the top `k` largest values encountered so far. If the heap size is less than `k`, we simply add the new value. Otherwise, we replace the smallest value in the heap with the new value if it is larger, updating the sum accordingly.
3. **Efficiency**: Sorting takes `O(n log n)` time. Each heap operation (insertion and extraction) takes `O(log k)` time, leading to an overall time complexity of `O(n log n + n log k)`, which is efficient for the problem constraints.

```python
import heapq
from typing import List

class Solution:
    def findMaxSum(self, nums1: List[int], nums2: List[int], k: int) -> List[int]:
        n = len(nums1)
        sorted_list = sorted((nums1[i], nums2[i], i) for i in range(n))
        
        ans = [0] * n
        heap = []
        sum_top = 0
        i = 0
        while i < n:
            j = i
            current_num = sorted_list[i][0]
            while j < n and sorted_list[j][0] == current_num:
                j += 1
                
            for idx in range(i, j):
                orig_idx = sorted_list[idx][2]
                ans[orig_idx] = sum_top
                
            for idx in range(i, j):
                num_val = sorted_list[idx][1]
                if len(heap) < k:
                    heapq.heappush(heap, num_val)
                    sum_top += num_val
                else:
                    if num_val > heap[0]:
                        sum_top -= heapq.heappop(heap)
                        heapq.heappush(heap, num_val)
                        sum_top += num_val
            
            i = j
        
        return ans
```

To solve this problem, we need to efficiently compute for each index `i` in the arrays `nums1` and `nums2` the sum of the top `k` values from `nums2` at indices `j` where `nums1[j] < nums1[i]`. Given the constraints where `n` can be as large as $10^5$, a brute-force approach would be infeasible. Instead, we leverage sorting and a min-heap to achieve an optimal solution.

### Approach
1. **Problem Analysis**: For each index `i`, we need to consider all indices `j` such that `nums1[j] < nums1[i]`. From these indices, we select at most `k` values from `nums2` that maximize the sum.
2. **Intuition**: By sorting the indices based on `nums1`, we can process each index in increasing order of `nums1`. For any index `i` during processing, all previously processed indices (with smaller `nums1` values) are candidates for the top `k` values in `nums2`.
3. **Algorithm Selection**: 
   - **Sorting**: First, we create a list of tuples `(nums1[i], nums2[i], i)` sorted by `nums1` in ascending order.
   - **Group Processing**: We process the sorted list in groups where each group consists of indices with the same `nums1` value. For each group:
     - **Set Answers**: The current sum of the top `k` values (from the min-heap) is assigned to all indices in the group (since these indices have `nums1` values equal to the current group's value, and no indices with smaller `nums1` have been processed yet).
     - **Update Heap**: We then add the `nums2` values of the current group to a min-heap. The heap maintains the top `k` largest values encountered so far, and we update the sum accordingly.
4. **Complexity Analysis**: 
   - **Sorting**: $O(n \log n)$ time.
   - **Heap Operations**: Each insertion and extraction operation on the heap takes $O(\log k)$ time. Since we process each element once, the total time is $O(n \log k)$.
   - **Overall Complexity**: $O(n \log n + n \log k)$, which is efficient for $n \leq 10^5$.

```python
import heapq
import itertools
from typing import List

class Solution:
    def findMaxSum(self, nums1: List[int], nums2: List[int], k: int) -> List[int]:
        n = len(nums1)
        arr = [(nums1[i], nums2[i], i) for i in range(n)]
        arr.sort(key=lambda x: x[0])
        
        heap = []
        current_sum = 0
        ans = [0] * n
        
        for key, group in itertools.groupby(arr, key=lambda x: x[0]):
            for item in group:
                idx = item[2]
                ans[idx] = current_sum
            
            group_nums2 = [item[1] for item in group]
            for num in group_nums2:
                if len(heap) < k:
                    heapq.heappush(heap, num)
                    current_sum += num
                else:
                    if num > heap[0]:
                        removed = heapq.heapreplace(heap, num)
                        current_sum = current_sum - removed + num
        return ans
```

### Explanation
1. **Initialization**: We start by creating a list of tuples containing `nums1[i]`, `nums2[i]`, and the index `i` for each element.
2. **Sorting**: The list is sorted based on `nums1` in ascending order to facilitate processing indices in increasing order of `nums1`.
3. **Group Processing**: Using `itertools.groupby`, we process the sorted list in groups where each group contains indices with the same `nums1` value.
4. **Answer Assignment**: For each index in the current group, we assign the current sum of the top `k` values (from the min-heap) as the answer. This sum represents the maximum sum achievable from `nums2` values of indices with `nums1` values strictly less than the current group's `nums1` value.
5. **Heap Maintenance**: After processing all indices in the current group, we add their `nums2` values to a min-heap. The heap ensures we only keep the top `k` values, updating the current sum whenever a larger value is encountered.
6. **Result Construction**: The answers for all indices are constructed and returned in the order corresponding to the original indices.

This approach efficiently handles the constraints by leveraging sorting and heap operations to minimize the time complexity, making it suitable for large input sizes.
